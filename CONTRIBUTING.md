
# Contributing to TargetJS

First of all, thank you for your interest in contributing to TargetJS!

Whether you're reporting a bug, suggesting an idea, improving the documentation, or submitting code, your help is greatly appreciated.

> **Note:** TargetJS is currently maintained by a single developer. While I review every contribution, responses may occasionally take a little time. 
Thank you for your patience and for helping improve the project.

## Reporting Bugs

Before opening an issue, please:

- Check whether the issue has already been reported.
- Try to reproduce it with the latest version of TargetJS.
- If possible, provide a minimal example that reproduces the problem.

When reporting a bug, please include:

- TargetJS version
- Browser and version
- Operating system
- Steps to reproduce
- Expected behavior
- Actual behavior

## Suggesting Features

Feature requests are welcome.

Please open a GitHub Issue describing:

- The problem you're trying to solve
- Your proposed solution
- Any alternatives you've considered

Discussion before implementation is encouraged.

## Pull Requests

Pull requests are always welcome.

Please try to:

- Keep each pull request focused on a single change.
- Follow the existing coding style.
- Update documentation when behavior changes.
- Add or update examples when appropriate.
- Keep commits clean and descriptive.

If you're planning a large change, consider opening an issue first so we can discuss the design.

## Documentation

Documentation improvements are just as valuable as code contributions.

If you notice unclear explanations, missing information, or typos in README.md, feel free to submit a pull request.

## Questions

If you're unsure about anything, feel free to open an issue. We'd rather answer questions early than have contributors spend time heading in the wrong direction.

Thank you for helping make TargetJS better!
